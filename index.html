About Project

In this coursework you are required as an individual, to design, build, test and evaluate a prototype web application that employs the technologies you will learn throughout the module. You are asked to build a Voter Registration web application for user registration and authentication, search and verification, adding, updating voter information, council wise segmentation, notifications and reminders.

The application is required to have more than one role and the roles are user, election officer and admin. Voters can register by providing personal details. Verify user identity during login. Users can search for their names in the voter list and verify their registration status. Display information such as name, address, and polling station. Election officials can add new voters or update existing voter details. Organize voters by councils,and polling stations. Send notifications about upcoming electionsand po ling dates. Announcements like Broadcast important messages to voters. The final functionality and specification of the web application you create is up to you.

As a minimum your application is likely to have 4 interface pages to allow users: to log in; to view a group of voters, to view an individual voter, and to add a voter. But it may have more depending on the functionality you decide to include. Note you may use standard templates, libraries, or existing APIs to support your design, but they must be acknowledged in your final report.

The web application must contain the following features:

  1. The application must contain both front end (client-side) and back end (server-side) code.
  2. More than one user role, e.g. Voter, Election officer and Admin
  3. A login page for voter/Election officer.
  4. Pages to view and select voters.
  5. A system allowing voter or election officer to input details that can be stored and recalled from a database.

The web application should contain the following features:

  1. A file upload system to allow images to be added to the application
  2. Voters should be able to manage (edit and update) their own details.
  3. Election officers should be able to view the voters and filter them by location and category
  4. A user with an Administrator role should be able to login to gain access to the system.
  5. Administrators should be able to review and manage all voters on the system

The web application could contain the following features:

  1. An administrator panel showing system information e.g. number of voters
  2. A display of voters/users that shows them at location on a map view
  3. Improved code implementation to mitigate against common security issues.
  4. Send notifications about upcoming elections and polling dates that are important to voters

You are encouraged to include additional functionality beyond the core functions as described in the must and should categories above.
